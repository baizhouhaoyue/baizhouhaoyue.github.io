第一章 类和对象的基本概念

1.1 C 和 C++中 struct 区别

核心思想：把一些数据进行封装来描述一个事物

c 语言 struct 只有变量。

c++语言 struct 既有变量，也有函数。

缺点：结构体成员对外部都是可见的。

1.2 类的封装

将数据和方法 封装在一起 加以权限区分 只能借助公共方法访问私有数据

使用关键字class定义一个类。

类的权限：private 私有 protected保护 public公有

注意：

1、类的内部 不区分权限， 类的外部才有权限之分。

2、如果类没有涉及到继承private和protected没有任何区别（都是私有特性）。

3、如想访问类内部的私有属性，可通过公共方法来实现。

案例1：定义及使用一个类（在类中实现类函数）

用户只能同过 公共方法 间接 操作 私有数据。

1.2 设计类的思想

1、明白该类涉及哪些数据（属性）。

2、 实现类当中公共方法。

1.3、设计一个Person类

1.4C++中结构体和类的区别

1.class 默认访问权限为 private,struct 默认访问权限为 public。

2.class访问控制，现实事物本身有些属性和行为是不对外开放。

第四章 动态对象创建

5.1.动态内存分配的差异

C语言中动态内存分配函数：malloc、calloc、realloc和free。

以malloc和free为例：

在C++中该系列函数：

1.malloc只分配内存空间，不会调用对象的初始化函数。

2.free只会释放内存空间，不会调用对象的析构函数。

故在C++中动态内存的分配引出了new和delete函数。

5.2 new和delete创建对象

new：分配空间，同时也会初始化该对象。

delete: 释放空间，同时会调用该对象的析构函数。

5.3 new和delete函数使用

new申请堆区空间，delete释放堆区空间。

8.1深拷贝与浅拷贝的概念

浅拷贝：同一类型的对象之间可以赋值，使得两个对象的成员变量的值相同，两个对象仍然是独立的两个对象。

注意：一般情况下， 浅拷贝没有任何副作用，但是当类中有指针，并且指针指向动态分配的内存空间，析构函数做了动态内存释放的处理，会导致内存问题。

深拷贝：当类中有指针，并且此指针有动态分配空间（在堆区分配了空间），析构函数做了释放处理，往往需要自定义拷贝构造函数，自行给指针动态分配空间

第九章 静态成员

9.1 静态数据成员

9.1.1 静态数据成员的定义及使用

1.静态数据成员属于类，不属于对象。

2.静态数据成员在所有的对象中共享。

\3. 静态成员变量必须在类中声明，在类外定义，在定义时在数据成员前不要加static。

9.1.2 静态数据成员的大小

在类当中，静态数据成员不占用类的空间大小。

9.1.3 访问私有的静态数据成员可通过静态成员函数或对象的普通成员函数

9.2静态成员函数

注意：静态成员函数主要为了访问静态变量，但是，不能访问普通成员变量。

静态成员函数的意义，不在于信息共享，数据沟通，而在于管理静态数据成员，完成对静态数据成员的封装。

普通成员函数可访问静态成员变量、也可以访问非静态成员变量。

扩展知识：

静态常量数据成员，同样属于类，对象之间共享，具有只读属性。
